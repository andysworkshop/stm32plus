Detailed Description

template<class C>
class stm32plus::scoped_array< C >

scoped_array<C> is like scoped_ptr<C>, except that the caller must allocate with new [] and the destructor deletes objects with delete [].

As with scoped_ptr<C>, a scoped_array<C> either points to an object or is nullptr. A scoped_array<C> owns the object that it points to. scoped_array<T> is thread-compatible, and once you index into it, the returned objects have only the threadsafety guarantees of T.
